# 页面设计说明（Desktop-first）：浅色系极简风全站改版

## 全局设计（适用于全站/所有区块）

### Layout
- 桌面优先：内容最大宽度 1080–1200px，左右留白增强阅读。
- 主要采用 Flex + Grid 混合：大区块纵向堆叠；卡片/图片区域使用 CSS Grid。
- 间距体系：8px 基准（8/16/24/32/48/64）。

### Meta Information
- Title：陆机阳｜AI运营创作者 & 金融数据分析师
- Description：个人介绍、能力优势、项目与经历、联系方式。
- Open Graph：
  - og:title 同 Title
  - og:description 同 Description
  - og:image 使用你的头像或一张代表性照片（浅色背景更佳）

### Global Styles（Design Tokens 建议）
- 背景：
  - Page bg：#F8FAFC（近白）
  - Section alt bg：#FFFFFF
- 文字：
  - 主标题：#0F172A
  - 正文：#334155
  - 次要信息：#64748B
- 边框/分隔：#E2E8F0（1px）
- 强调色（单一品牌色，克制使用）：#2563EB
- 卡片：白底 + 1px 边框（优先），阴影尽量弱（可选：shadow-sm）。
- 圆角：12px（卡片），9999px（胶囊按钮/导航）。
- 字体与层级（桌面）：
  - H1 44–52 / 700
  - H2 28–32 / 650
  - H3 18–20 / 600
  - Body 15–16 / 400–500
  - Caption 12–13 / 400
- 按钮：
  - Primary：品牌色底 + 白字；hover 轻微加深；focus 显示 outline
  - Secondary：白底 + 边框；hover 背景 #F1F5F9
- 链接：默认品牌色；hover 下划线或颜色加深。
- 动效：保持“少而精”，过渡 150–220ms；避免大面积浮夸光效。

## 页面：首页（单页滚动）

### Page Structure
从上到下的堆叠结构（现有区块不变）：
1) Header（固定/吸顶）
2) Hero（home）
3) About（about）
4) Advantages（advantages）
5) 新增：照片展示（不改变原有区块顺序时，可插入在 Advantages 与 Projects 之间；且不强制加入顶部导航项）
6) Projects（projects）
7) Experience（experience）
8) Contact（contact）
9) Footer

### Sections & Components

#### 1) Header（现有）
- 结构：左侧姓名/标识（如有）+ 右侧胶囊导航（各锚点）。
- 样式：白底或半透明白（blur 可选），1px 底部分隔线；active 状态用品牌色小圆点/下划线（保持克制）。
- 交互：滚动高亮与点击平滑滚动保持现有逻辑。

#### 2) Hero（现有）
- 布局：左右双栏（桌面）
  - 左：姓名、title、subtitle、简介摘要、CTA 按钮
  - 右：头像/形象卡片（如现有）
- 视觉：去除深色背景与高对比装饰；改用浅色大留白 + 少量细线/轻渐变点缀（可选）。
- CTA：主按钮（查看项目）+ 次按钮（联系我）并排。

#### 3) About（现有）
- 布局：单列文本为主，关键字段（学校/专业/地点等）用两列信息清单或浅色卡片。
- 视觉：强调可读性（行高 1.6–1.8），段落间距明确。

#### 4) Advantages（现有）
- 布局：优势要点用卡片列表；技能条（如有）使用细条 + 品牌色填充。
- 视觉：减少阴影，边框与留白体现层级；图标统一线性风格，颜色使用次要文字色。

#### 5) 新增：照片展示版块
- 目的：提供可放照片的承载位，适配你的摄影/形象内容。
- 标题：建议使用中性标题（例如“照片 / Photography”）；若你坚持“文案不变”，则标题可隐藏，仅展示图片墙。
- 布局（桌面）：
  - Grid：3 列（>=1200px），间距 16–20px；卡片圆角 14px；统一浅灰边框。
  - 图片比例：建议统一 4:3 或 3:4；支持 object-fit: cover，避免高度跳变。
- 布局（移动）：1 列或 2 列自适应。
- 交互（二选一）：
  1) 点击图片打开原图新窗口（实现最轻）
  2) 点击打开轻量预览（遮罩层 + 左右切换 + ESC 关闭）
- 状态：加载前占位（灰色骨架块），图片加载失败显示“占位图 + 重试/打开原链接”。

#### 6) Projects（现有）
- 布局：卡片列表/网格（保持现有信息结构）；标签使用浅底 chip（#F1F5F9）+ 深色字。
- 交互：卡片 hover 仅轻微上浮/描边加深，避免厚重阴影。

#### 7) Experience（现有）
- 布局：时间线使用细线（#E2E8F0）+ 节点圆点（品牌色/深色）；内容区块用白卡。
- 可读性：标题更突出，描述使用次要文字色，段落间距 12–16。

#### 8) Contact（现有）
- 布局：两列（桌面）
  - 左：一句话引导 + 联系方式
  - 右：社交入口（Github/Email/WeChat）卡片
- 交互：mail/外链 hover 统一；微信 detail 若为弹出/tooltip，采用浅色浮层。

#### 9) Footer（现有）
- 视觉：浅灰背景（#F1F5F9）或白底 + 上分隔线；文字使用次要色。

### Responsive Behavior
- >=1024px：双栏与 3 列图片网格为主
- 768–1023px：部分区块降为单栏，图片网格 2 列
- <768px：全部单栏；图片网格 1–2 列；Header 胶囊导航可横向滚动
